DDNS: When the device works in Router or WISP mode, dynamic DNS (DDNS) maps a fixed domain name to a dynamic WAN IP address.

To use this function, a DDNS account registered at the DDNS service providers supported by this device is required.

User Name/Password/Domain Name: Enter the user name, password and domain name that you registered with DDNS service provider to enable the DDNS service of the device.

Remote Web Management: With this function enabled, you can visit the web UI of this device remotely through its WAN port.

IP Address: IP address of the computer allowed to manage this device remotely. "All" indicates that any computer in WAN can manage this device remotely.

IP Address: enter the IP address of the computer allowed to manage this device remotely.

Port: port number used for remote management of device. Default: 8080. You can change it if necessary.

Reboot Schedule: you can create a reboot schedule for this device here. If this function is enabled, verify that the device is connected to the internet and the system time of the device is synchronized with the GMT time through the internet to ensure proper operation of the function.

Login Timeout Interval: you can set the login timeout interval of this device here. If you perform no operation within the interval after logging in to the device, the system logs you out.

SNMP Agent: whether to enable the SNMP Agent function of the device. With this function enabled, you can manage the device using SNMP software.

Read Community: Read password used between an NMS and the SNMP agent. The SNMP agent allows the NMS to use the Read community to read variables in the MIB of this device.

Read/Write Community: Read/Write password used between an NMS and the SNMP agent. The SNMP agent allows the NMS to use the Read/Write community to read/write variables in the MIB of this device.

Ping Watch Dog: Periodically ping target IP address to check the network connectivity and identify whether the device malfunctions. If it malfunctions, the device will reboot automatically to ensure the network performance.

IP Address: allow you to set the IP Address of target host.

Ping Interval: interval at which the device sends Ping packets. Default: 300 s

Ping Startup Delay: interval at which the device enables the Ping Watch Dog function after the device reboots. Default: 300 s.

Threshold of lost packets for reboot: specify the threshold of lost packet that triggers reboot. If N is set, the device will reboot automatically when it sends N Ping packets to target IP address/domain name, and does not receive response. Range: 1 to 65535, default: 3.

DMZ Host: If a host in LAN is set to DMZ host, the host exposes itself to WAN, and can achieve bidirectional communication with external network nodes.

DMZ Host IP Address: Enter IP address of a computer to be set as DMZ host in LAN.

Telnet Service: Used to enable or disable Telent function.

UPnP: UPnP protocol allows the device to identify applications supported UPnP in LAN automatically, and enable ports for these applications to achieve automatic port mapping function.
