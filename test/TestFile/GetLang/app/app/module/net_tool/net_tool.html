Site Survey: It is used to scan WiFi signals nearby for analysis with SSIDs, MAC addresses, channels and signal strength marked.

Ping: You can use ping to detect the connectivity and quality of network connection.

Ping Packet: set the size of transmitted ping packets.

Packet size: specify the size of the packet transmitted by ping.

Destination IP Address/Domain Name: enter the IP address or domain name of destination host.

Traceroute: You can use the Traceroute tool to detect the routes that the packets pass by from the device to destination host.

Destination IP Address/Domain Name: enter the IP address or domain name of destination host.

Speed Test: used to test the throughput between two Tenda CPE in the same network. The test requires one of the two devices to be set as a server and the other as a client. The client launches the test request to the server and the server responds to it. The test result displays on the "Speed Test" page of the client.

Client: the side that launches the test request. You need to set speed test parameters on client side, including IP address of peer device, speed test direction, and so on.

Server: On the server end, click "Speed Test" button on "Diagnose" page to enable this function.

IP Address of Peer AP: IP address of peer bridge. You can select an existing IP address, or enter one.

IP Address: Enter the IP address of peer bridge.

HTTP Port: port number of HTTP service. Default: 80

User Name: user name of web UI of peer device.

Password: password of web UI of peer device.

No. of Test Connection: specify the number of test connection launched by the client. Range: 1 to 20.

Direction: Select the test speed direction. RX (Receive): only test the speed that the peer device transmits data to this device. TX (Transit): only test the speed that this device transmits data to peer device.

Time: period of speed test.

Test Progress: process of speed test.
